# Health Data Processing — Client Project

This is a health data processing project for a single client, designed to safely handle sensitive medical documents by separating PII-containing data from anonymised data.

Processing scripts live in the shared `../health_process/` repository. Activate its virtual environment before running scripts:

```bash
source ../health_process/venv/bin/activate
```

## Project Structure

```text
health_<client>/
├── contains_pii/
│   ├── 0_raw_inbox/
│   │   ├── nhs_gp/                 ← NHS SAR PDF — CONTAINS PII
│   │   └── other/                  ← Dental, private, wearables — CONTAINS PII
│   └── 1_extracted_text/           ← Auto-generated by scripts — CONTAINS PII
├── no_pii/
│   ├── 2_unstructured/             ← PII-redacted text — SAFE
│   ├── 3_structured/               ← FHIR JSON resources — SAFE
│   └── 4_ai_outputs/               ← Analysis outputs — SAFE
├── pii_config.json                 ← CONTAINS PII — DO NOT ACCESS
├── page_config.json                ← SAR page ranges (from template)
├── pii_config.template.json        ← Template for PII config
├── page_config.template.json       ← Template for page ranges
├── PROCESS_LOG.md                  ← Processing audit trail
└── scripts -> ../health_process/scripts/
```

### Source type subfolders (in 0_raw_inbox/)

- **`nhs_gp/`** — NHS Subject Access Request outputs, GP records. Processed with `extract_sar.py` (single large PDF, 3-fork pipeline with OCR).
- **`other/`** — Dental, private providers, wearables, standalone documents. Processed with `extract_batch.py` (multiple simple PDFs, embedded text only).

## Configuration Files

Create from templates before running extraction:

| Template | Copy to | Purpose |
|----------|---------|---------|
| `pii_config.template.json` | `pii_config.json` | PII values to redact (names, DOB, phones, postcodes, etc.) |
| `page_config.template.json` | `page_config.json` | SAR PDF page ranges for 3-fork split |

Both `pii_config.json` and `page_config.json` are gitignored.

## Quick Commands

**NHS SAR extraction (3-fork pipeline):**

```bash
python3 scripts/extract/extract_sar.py contains_pii/0_raw_inbox/nhs_gp/<pdf_file>
```

**Batch extraction (other/ PDFs):**

```bash
python3 scripts/extract/extract_batch.py
```

**PII removal (single file):**

```bash
python3 scripts/pii/strip_pii.py <input_file> <output_dir>
```

**PII removal (batch):**

```bash
python3 scripts/pii/strip_pii.py --batch contains_pii/1_extracted_text/other/ no_pii/2_unstructured/
```

**FHIR structuring (interactive):**

```bash
python3 scripts/fhir/structure_sar.py <DOC_NAME> --mode interactive
```

**FHIR validation:**

```bash
python3 scripts/fhir/validate_fhir.py no_pii/3_structured/
```

## Processing Pipeline

The pipeline follows numbered stages. Files stay in place after processing — `PROCESS_LOG.md` records what has been done.

1. **Raw Intake** (`contains_pii/0_raw_inbox/nhs_gp/` or `other/`) — Place raw medical documents here
2. **Text Extraction** (`contains_pii/1_extracted_text/`) — Auto-generated by extraction scripts
3. **PII Redaction** → Output to `no_pii/2_unstructured/` after redaction
4. **FHIR Structuring** (`no_pii/3_structured/`) — Convert to FHIR format via `structure_sar.py` or Claude Code
5. **AI Analysis** (`no_pii/4_ai_outputs/`) — AI processing and analysis outputs

### NHS SAR extraction (3-fork pipeline)

1. Place SAR PDF in `contains_pii/0_raw_inbox/nhs_gp/`
2. Fill in `pii_config.json` with the client's PII
3. Fill in `page_config.json` by reviewing the PDF structure
4. Run extraction:

```bash
python3 ../health_process/scripts/extract/extract_sar.py contains_pii/0_raw_inbox/nhs_gp/<pdf_file>
```

This runs pre-processing (watermark removal, PII removal from PDF, split into 3 PDFs) then up to 3 extraction forks:

- **Fork 1** (extract_text): PyMuPDF for pages before attachments
- **Fork 2** (ocr_text): Apple Vision OCR for attachment pages (excl. Lloyd George)
- **Fork 3** (transcribe_text): manual workflow for Lloyd George handwritten pages

Output: up to 3 PII-redacted `.md` files in `no_pii/2_unstructured/`

5. Run FHIR structuring:

```bash
python3 ../health_process/scripts/fhir/structure_sar.py <DOC_NAME> --mode interactive
```

### Other documents (batch extraction)

```bash
python3 ../health_process/scripts/extract/extract_batch.py
```

Extracts embedded text from all PDFs in `contains_pii/0_raw_inbox/other/` to `contains_pii/1_extracted_text/other/`, then PII-redact with:

```bash
python3 ../health_process/scripts/pii/strip_pii.py --batch contains_pii/1_extracted_text/other/ no_pii/2_unstructured/
```

- `PROCESS_LOG.md` records all processing actions for audit trail
